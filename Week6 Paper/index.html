The Power Behind GitHub

Today, GitHub stands at the forefront of the open-source community as well as providing people to work together on projects seamlessly. But how does GitHub process all the requests it gets, much less store all the repositories that are created daily?

GitHubs's Infrastructure

When you send a request to GitHub.com, a load-balancer will send you off to their frontend machines. These machines are 8 core, 16GB RAM bare metal servers. If you happen to search through GitHub, they will send the request to 8 core, 32GM RAM bare metal servers, with 15k RPM SAS drives. These machines will process and return results for whatever your query was.

Now, most people use GitHub to store their projects and data. These servers are 8 core, 16GB RAM bare metal servers, with six 300GB 15K RPM SAS drives arranged in RAID 10. In case of a server failure, these servers are paired with one another. Should one fail, the other will simply pick up where the other left off. As an additional safety measure, all storage is replicated between the paired servers.

With how these separate tiers of servers are setup, it is easy for GitHub to upscale and down scale on their servers. With this setup, GitHub is able to achieve site response times that are consistently under 100ms. With how popular Github is getting these days, its good for such a setup to be able to expand.

GitHubs's Security

GitHub initiates a SSH connection with you and their servers the moment you load the page. To be able to do this, GitHub stores your key into their database, and performs actions that are based on your key. Once you Github has located your key and established a connection, the backend file servers are able to talk with your Git requests, and make sure that your files are uploaded securely.

All files are backed up and stored on at least three different locations, as well as an off-site backup incase the main data-centers are destroyed. In addition, Github also provides a dedicated firewall and VPN services on their servers to block unauthorized access. DDoS attacks are becoming more common these days, and as such GitHub provides DDoS mitigation services to help prevent the site from being taken down.
